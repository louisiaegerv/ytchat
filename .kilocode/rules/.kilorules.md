# .kilorules.md

> **Primary UI Framework: Shadcn UI**
>
> 1. **Check First:** Before generating any UI code, search your internal knowledge and use the `context7` MCP tool to identify the relevant Shadcn component.
> 2. **Automatic Installation:** If a required Shadcn component is missing from `@/components/ui`, use the `shadcn` MCP tool to run `npx shadcn add [component-name]` before writing the implementation code.
> 3. **Theming & Design System:** Strictly adhere to the project's Tailwind CSS variables (e.g., use `text-primary`, `bg-background`, `border-input`). Do not hardcode hex codes.
> 4. **Component Composition:** Favor Shadcn primitives over raw HTML/Tailwind. For example, use the `Button` component from `@/components/ui/button` instead of a `<button>` tag with Tailwind classes.
> 5. **Imports:** Always use the `@/` path alias for imports (e.g., `import { Button } from "@/components/ui/button"`).
> 6. **Forms:** When creating forms, always use the Shadcn `Form` component (which integrates `react-hook-form` and `zod`) rather than standard HTML forms.
